# Phase 12, Plan 1: Persistent Topic Core

## Objective

Replace ad-hoc log usage with a first-class `BayouTopic<M>` — a durable, log-backed topic that
survives process restarts. `system.topic(name, serializer)` returns a `BayouTopic<M>` (idempotent).
`topic.publish(M)` appends to the gumbo log AND delivers to all live subscribers. `topic.subscribe`
/ `topic.unsubscribe` manage live delivery. The existing `BayouPubSub` (`system.pubsub()`) is
unchanged — `BayouTopic` is the durable alternative.

## Context

**Current state:** Bayou has `BayouPubSub` (in-memory, `system.pubsub()`) for ephemeral pub/sub.
All four actor flavors work. Gumbo log access follows the pattern in `EventSourcedActorRunner`:
`system.sharedLog().getView(LogTag.of(namespace, key))` → `LogView`;
`logView.append(bytes).join()`; `logView.readAll().join()` → `List<LogEntry>`.

**Key design decisions:**

- `BayouTopic<M>` is a public facade in `com.cajunsystems.bayou`. It holds a `Ref<TopicCommand<M>>`
  to its internal actor and delegates `publish/subscribe/unsubscribe` as fire-and-forget tells.
- `TopicCommand<M>` is a package-private sealed interface with three records:
  `Publish<M>(M payload)`, `Subscribe<M>(Ref<M> subscriber)`, `Unsubscribe<M>(Ref<M> subscriber)`.
- `TopicActor<M>` is a package-private class implementing `Actor<TopicCommand<M>>`. It owns the
  `LogView` at `LogTag.of("bayou.topic", name)`, maintains `List<Ref<M>> liveSubscribers`
  (CopyOnWriteArrayList for thread safety within the actor thread).
- On `Publish`: append serialized bytes to log view, then `tell` each live subscriber; skip dead
  ones and remove lazily (same pattern as `BayouPubSub`).
- On `Subscribe`: add `Ref<M>` to `liveSubscribers` (no replay — that's Phase 14).
- On `Unsubscribe`: remove from `liveSubscribers`.
- `BayouSystem` gets `private final ConcurrentHashMap<String, BayouTopic<?>> topics` and
  `public <M> BayouTopic<M> topic(String name, BayouSerializer<M> serializer)` — idempotent:
  first call creates, subsequent calls return cached instance.
- Topic's internal actor is registered in `BayouSystem.actors` with ID `"bayou-topic-" + name`
  so the existing `shutdown()` loop stops it automatically — no special teardown needed.
- `TopicActor` initializes `logView` in its constructor (before `preStart`), making it available
  from the first message. No replay on startup in Phase 12 — that is Phase 14.
- **Test restart simulation:** use the same `SharedLogService` (and thus same
  `InMemoryPersistenceAdapter`) across two sequential `BayouSystem` instances. After system 1
  shuts down and system 2 starts up, verify persistence by counting entries in
  `logView.readAll()` — NOT by subscriber receipt (replay is Phase 14).

## Tasks

### Task 1 — TopicCommand + BayouTopic + TopicActor + BayouSystem.topic()

**Create `src/main/java/com/cajunsystems/bayou/TopicCommand.java`:**

```java
package com.cajunsystems.bayou;

sealed interface TopicCommand<M> permits TopicCommand.Publish, TopicCommand.Subscribe, TopicCommand.Unsubscribe {
    record Publish<M>(M payload) implements TopicCommand<M> {}
    record Subscribe<M>(Ref<M> subscriber) implements TopicCommand<M> {}
    record Unsubscribe<M>(Ref<M> subscriber) implements TopicCommand<M> {}
}
```

**Create `src/main/java/com/cajunsystems/bayou/TopicActor.java`:**

```java
package com.cajunsystems.bayou;

import com.cajunsystems.bayou.actor.Actor;
import com.cajunsystems.gumbo.LogView;
import com.cajunsystems.gumbo.LogTag;

import java.util.concurrent.CopyOnWriteArrayList;
import java.util.List;

class TopicActor<M> implements Actor<TopicCommand<M>> {

    private final String name;
    private final BayouSerializer<M> serializer;
    private LogView logView;
    private final List<Ref<M>> liveSubscribers = new CopyOnWriteArrayList<>();

    TopicActor(String name, BayouSerializer<M> serializer) {
        this.name = name;
        this.serializer = serializer;
    }

    @Override
    public void preStart(BayouContext<TopicCommand<M>> ctx) {
        logView = ctx.system().sharedLog().getView(LogTag.of("bayou.topic", name));
    }

    @Override
    @SuppressWarnings("unchecked")
    public void handle(TopicCommand<M> cmd, BayouContext<TopicCommand<M>> ctx) {
        switch (cmd) {
            case TopicCommand.Publish<M> p -> {
                try {
                    logView.append(serializer.serialize(p.payload())).join();
                } catch (Exception e) {
                    ctx.logger().error("Failed to persist message to topic '{}'", name, e);
                }
                liveSubscribers.removeIf(ref -> {
                    if (!ref.isAlive()) return true;
                    try { ref.tell(p.payload()); } catch (Exception ignored) {}
                    return false;
                });
            }
            case TopicCommand.Subscribe<M> s -> liveSubscribers.add(s.subscriber());
            case TopicCommand.Unsubscribe<M> u -> liveSubscribers.remove(u.subscriber());
        }
    }
}
```

**Create `src/main/java/com/cajunsystems/bayou/BayouTopic.java`:**

```java
package com.cajunsystems.bayou;

/**
 * A durable, log-backed topic. Messages published to a topic are appended to the gumbo log
 * and delivered to all live subscribers. Topic history survives process restarts.
 *
 * <p>Obtain via {@link BayouSystem#topic(String, BayouSerializer)}.
 */
public class BayouTopic<M> {

    private final Ref<TopicCommand<M>> actorRef;

    BayouTopic(Ref<TopicCommand<M>> actorRef) {
        this.actorRef = actorRef;
    }

    /** Appends {@code message} to the log and delivers to all live subscribers. */
    public void publish(M message) {
        actorRef.tell(new TopicCommand.Publish<>(message));
    }

    /** Registers a live subscriber. Messages published after this call are delivered. */
    public void subscribe(Ref<M> subscriber) {
        actorRef.tell(new TopicCommand.Subscribe<>(subscriber));
    }

    /** Removes a live subscriber. */
    public void unsubscribe(Ref<M> subscriber) {
        actorRef.tell(new TopicCommand.Unsubscribe<>(subscriber));
    }
}
```

**Modify `src/main/java/com/cajunsystems/bayou/BayouSystem.java`:**

- Add field: `private final ConcurrentHashMap<String, BayouTopic<?>> topics = new ConcurrentHashMap<>();`
- Add public method:

```java
@SuppressWarnings("unchecked")
public <M> BayouTopic<M> topic(String name, BayouSerializer<M> serializer) {
    return (BayouTopic<M>) topics.computeIfAbsent(name, k -> {
        String actorId = "bayou-topic-" + k;
        TopicActor<M> actor = new TopicActor<>(k, serializer);
        Ref<TopicCommand<M>> ref = spawn(actorId, actor);
        return new BayouTopic<>(ref);
    });
}
```

### Task 2 — PersistentTopicTest

**Create `src/test/java/com/cajunsystems/bayou/PersistentTopicTest.java`:**

Tests to write (use the same `InMemoryPersistenceAdapter` instance across system restarts):

1. **`publishDeliversToLiveSubscribers`** — subscribe 2 actors, publish 3 messages, assert all 6
   deliveries received (Awaitility).

2. **`unsubscribeStopsDelivery`** — subscribe, publish 1, unsubscribe, publish 1 more, assert only
   first message received.

3. **`topicIsIdempotent`** — call `system.topic("news", ser)` twice, assert same `BayouTopic`
   instance returned (reference equality).

4. **`publishPersistsToLog`** — publish 10 messages on system 1, shut it down, create system 2
   with the same `SharedLogService`, call `system2.sharedLog().getView(LogTag.of("bayou.topic", "events")).readAll().join()`,
   assert entry count is 10.

5. **`topicIsolation`** — publish to `"topic-a"` and `"topic-b"`, assert subscriber on `"topic-a"`
   only receives `"topic-a"` messages.

6. **`deadSubscriberIsCleanedUp`** — subscribe actor, stop it, publish — assert no exception and
   the dead ref is silently skipped.

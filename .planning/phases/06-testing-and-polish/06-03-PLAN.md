### Task 1 — Add supervision section to README.md

Append a `## Supervision` section after the existing content.

The section must cover these topics in order, each with a code example:

**1. Overview paragraph** — "let it crash" philosophy, how supervisors monitor children and apply
strategies.

**2. Basic supervisor** — `spawnSupervisor()` + `SupervisorActor` interface + `ChildSpec` factory
methods (stateless, stateful, eventSourced) + `SupervisionStrategy` choice.

```java
SupervisorRef ref = system.spawnSupervisor("my-supervisor", new SupervisorActor() {

    @Override
    public List<ChildSpec> children() {
        return List.of(
            ChildSpec.stateless("worker",  (msg, ctx) -> process(msg)),
            ChildSpec.stateful("counter",  new CounterActor(), new JavaSerializer<>()),
            ChildSpec.eventSourced("ledger", new LedgerActor(), new JavaSerializer<>())
        );
    }

    @Override
    public SupervisionStrategy strategy() {
        return new OneForOneStrategy(new RestartWindow(5, Duration.ofSeconds(60)));
    }
});
```

**3. Supervision strategies** — `OneForOneStrategy` (restart only the crashed child) vs
`AllForOneStrategy` (restart all children). One-sentence explanation of each.

**4. Restart window + death spiral guard** — explain `RestartWindow(maxRestarts, within)` and
`RestartWindow.UNLIMITED`. When the window is exceeded, the supervisor escalates to its own
parent (or stops gracefully if top-level).

```java
// Restart up to 5 times within 60 s — then escalate:
new OneForOneStrategy(new RestartWindow(5, Duration.ofSeconds(60)))

// Always restart, no limit:
new OneForOneStrategy(RestartWindow.UNLIMITED)
```

**5. RestartDecision** — table of four decisions returned by `SupervisionStrategy.decide()`:
`RESTART`, `RESTART_ALL`, `STOP`, `ESCALATE`. Custom lambda strategy example.

```java
// Custom strategy: always stop instead of restart
SupervisionStrategy stopAll = (childId, cause) -> RestartDecision.STOP;
```

**6. Dynamic child spawning** — `SupervisorRef.spawnChild(ChildSpec)` for adding children at
runtime. Mention that `children()` defaults to empty list for dynamic-only supervisors.

```java
// Dynamic-only supervisor: no initial children
SupervisorRef sup = system.spawnSupervisor("router", new SupervisorActor() {
    @Override
    public SupervisionStrategy strategy() {
        return new OneForOneStrategy(RestartWindow.UNLIMITED);
    }
});

// Add children at runtime from any thread
sup.spawnChild(ChildSpec.stateless("worker-1", handler));
sup.spawnChild(ChildSpec.stateless("worker-2", handler));
```

**7. Nested supervisors** — `ChildSpec.supervisor()` for building supervision trees. Brief
explanation of how escalation propagates up.

```java
system.spawnSupervisor("root", new SupervisorActor() {
    @Override
    public List<ChildSpec> children() {
        return List.of(
            ChildSpec.supervisor("db-group", new SupervisorActor() {
                @Override
                public List<ChildSpec> children() {
                    return List.of(ChildSpec.stateless("db-writer", writerActor));
                }
                @Override
                public SupervisionStrategy strategy() {
                    return new OneForOneStrategy(new RestartWindow(3, Duration.ofSeconds(30)));
                }
            })
        );
    }
    @Override
    public SupervisionStrategy strategy() {
        return new OneForOneStrategy(RestartWindow.UNLIMITED);
    }
});
```

**8. Stopping a supervisor** — `ref.stop()` propagates to all children.

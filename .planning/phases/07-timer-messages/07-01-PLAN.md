# Phase 7, Plan 1: Timer Messages

## Objective

Add `scheduleOnce` / `schedulePeriodic` / `TimerRef.cancel()` to Bayou — the `Process.send_after` equivalent. Timers deliver messages into the actor's own mailbox as normal `M` messages; no special handling required in the actor loop.

## Context

**Key discoveries from codebase exploration:**

- `BayouContext` is currently **untyped** — no generic parameter. Must be made `BayouContext<M>` so `scheduleOnce(Duration, M message)` can be typed correctly.
- `BayouContextImpl` does **not** hold a `Ref` to its own actor. Must add `Ref<M> self` field, set in `AbstractActorRunner` constructor.
- `BayouSystem` has **no** `ScheduledExecutorService`. Must add one (single-threaded is sufficient — timers just `tell()` into mailboxes, no ordering concerns between actors).
- Mailbox loop polls with 100ms timeout. Timer-fired messages queue normally; latency is bounded by the poll interval (acceptable for actor timers).
- `AbstractActorRunner` owns `cleanup()` — the right place to cancel all actor-owned timers.

**Design decisions:**

- `BayouContext<M>` generic — add `M` type parameter. All existing actor interfaces (`Actor<M>`, `StatefulActor<S,M>`, `EventSourcedActor<S,M,E>`) update their `BayouContext` parameter to `BayouContext<M>`. Lambda users are unaffected.
- Timer messages are plain `M` values delivered via `self.tell(message)` — no wrapper type needed.
- `TimerRef` is a public interface (`cancel()`, `isCancelled()`). `TimerRefImpl` is package-private.
- `AbstractActorRunner` tracks `Set<TimerRefImpl> activeTimers` — cancelled atomically in `cleanup()`.
- `BayouSystem` exposes `scheduledExecutor()` package-private (not public API).

## Tasks

### Task 1 — Infrastructure

**Create `TimerRef` public interface and `TimerRefImpl` package-private class.**

`TimerRef.java` (public, in `com.cajunsystems.bayou`):
```java
public interface TimerRef {
    void cancel();
    boolean isCancelled();
}
```

`TimerRefImpl.java` (package-private):
```java
final class TimerRefImpl implements TimerRef {
    private final ScheduledFuture<?> future;
    private final AtomicBoolean cancelled = new AtomicBoolean(false);

    TimerRefImpl(ScheduledFuture<?> future) { this.future = future; }

    @Override public void cancel() {
        if (cancelled.compareAndSet(false, true)) future.cancel(false);
    }

    @Override public boolean isCancelled() {
        return cancelled.get() || future.isCancelled();
    }
}
```

**Add `ScheduledExecutorService` to `BayouSystem`:**
- Add field: `private final ScheduledExecutorService scheduledExecutor = Executors.newSingleThreadScheduledExecutor(r -> Thread.ofVirtual().name("bayou-timer").unstarted(r));`
- Add package-private accessor: `ScheduledExecutorService scheduledExecutor() { return scheduledExecutor; }`
- In `shutdown()`: call `scheduledExecutor.shutdownNow()` before closing the log.

**Make `BayouContext<M>` generic:**
- Change `BayouContext` to `BayouContext<M>`.
- Add methods: `TimerRef scheduleOnce(Duration delay, M message);` and `TimerRef schedulePeriodic(Duration period, M message);`
- Update all actor interfaces to `BayouContext<M>`:
  - `Actor<M>`: `void handle(M message, BayouContext<M> ctx)`
  - `StatefulActor<S,M>`: `S reduce(S state, M message, BayouContext<M> ctx)` + preStart/postStop if present
  - `EventSourcedActor<S,M,E>`: update `BayouContext<M>` parameter

**Add `Ref<M> self` to `BayouContextImpl`:**
- Add field `private Ref<M> self;` and setter `void setSelf(Ref<M> self) { this.self = self; }`
- Make `BayouContextImpl<M>` generic accordingly.

**Track timers in `AbstractActorRunner`:**
- Add field: `private final Set<TimerRefImpl> activeTimers = ConcurrentHashMap.newKeySet();`
- In `cleanup()`: `activeTimers.forEach(TimerRef::cancel); activeTimers.clear();` — add before existing cleanup body.

**Wire `self` reference:** In `AbstractActorRunner` subclasses (or base class), after the `Ref` is created in `BayouSystem.spawn*()`, call `runner.context.setSelf(ref)`. Find where runners are constructed in `BayouSystem` and inject the ref.

### Task 2 — Timer API Implementation

**Implement `scheduleOnce` and `schedulePeriodic` in `BayouContextImpl<M>`:**

```java
@Override
public TimerRef scheduleOnce(Duration delay, M message) {
    ScheduledFuture<?> future = system().scheduledExecutor().schedule(
        () -> self.tell(message),
        delay.toMillis(),
        TimeUnit.MILLISECONDS
    );
    TimerRefImpl ref = new TimerRefImpl(future);
    // Register with runner for cleanup on actor stop
    runner.activeTimers.add(ref);
    future.whenCompleteAsync((v, t) -> runner.activeTimers.remove(ref)); // cleanup after fire
    return ref;
}

@Override
public TimerRef schedulePeriodic(Duration period, M message) {
    ScheduledFuture<?> future = system().scheduledExecutor().scheduleAtFixedRate(
        () -> { if (!self.isAlive()) return; self.tell(message); },
        period.toMillis(),
        period.toMillis(),
        TimeUnit.MILLISECONDS
    );
    TimerRefImpl ref = new TimerRefImpl(future);
    runner.activeTimers.add(ref);
    return ref;
}
```

**Note on `whenCompleteAsync`:** `ScheduledFuture` does not support `whenCompleteAsync`. Use a wrapper lambda instead: wrap the timer action to remove from `activeTimers` after a one-shot fires.

**Cleaner approach for cleanup of fired one-shot timers:**
```java
ScheduledFuture<?> future = system().scheduledExecutor().schedule(
    () -> {
        self.tell(message);
        runner.activeTimers.remove(timerRef[0]); // remove self after delivery
    },
    delay.toMillis(), TimeUnit.MILLISECONDS
);
TimerRefImpl timerRef[] = { new TimerRefImpl(future) };
runner.activeTimers.add(timerRef[0]);
return timerRef[0];
```
Or use a holder object. The exact pattern is flexible — what matters is that fired one-shots don't accumulate in `activeTimers` forever.

**Access to `runner`:** `BayouContextImpl` needs a reference to its `AbstractActorRunner`. It already exists as `context` is a field of the runner — but the context doesn't hold the runner back. Add `private AbstractActorRunner<M> runner;` to `BayouContextImpl` and set it in the runner constructor.

### Task 3 — Tests

Create `src/test/java/com/cajunsystems/bayou/TimerTest.java`.

Use Awaitility (already a test dependency) for timing assertions. Do **not** use `Thread.sleep` for assertions.

**Test cases:**

1. **`scheduleOnce_firesAfterDelay`** — Actor receives exactly one timer message after the delay. Use a `CountDownLatch` or `AtomicInteger` counter. Assert count == 1 after a reasonable wait (e.g., Awaitility `within(500ms).untilAsserted`).

2. **`scheduleOnce_cancelPreventsDelivery`** — Actor calls `scheduleOnce(200ms, msg)`, immediately calls `timerRef.cancel()`. Assert actor never receives the message (wait 400ms, count stays 0).

3. **`schedulePeriodic_firesRepeatedly`** — Actor schedules periodic message every 50ms. Assert count reaches 3 within 500ms. Then call `timerRef.cancel()`. Assert count stops incrementing.

4. **`actorStop_cancelsActiveTimers`** — Actor schedules a 200ms timer, then is stopped immediately. Assert message is never received (wait 400ms). Verifies `cleanup()` cancels timers.

5. **`multipleTimers_independentCancellation`** — Actor schedules two timers (A at 100ms, B at 300ms). Cancel B immediately. Assert A fires, B does not.

Each test creates a fresh `BayouSystem`, spawns a stateless actor that increments a shared `AtomicInteger` when it receives the timer message, and asserts using Awaitility.

# Phase 8, Plan 2: Linking & trapExits

## Objective

Add bidirectional actor linking and `trapExits` to Bayou. When two actors are linked and one dies, the other receives a `LinkedActorDied` signal ŌĆö and unless it is trapping exits, it crashes as a consequence. This is the Erlang `Process.link` / `Process.flag(:trap_exit, true)` equivalent.

Depends on Plan 1: signal infrastructure and `Terminated` must already be in place.

## Context

**From Plan 1:**
- `AbstractActorRunner` has `signalListeners` (CopyOnWriteArrayList) and `signal(Signal)` method.
- `processSignalEnvelope(Signal)` currently just calls `handleSignal(signal)`.
- `LinkedActorDied.java` exists as a stub permit of `Signal`.
- `BayouSystem` has a `runners` map for looking up runners by actorId.

**Design decisions for linking:**

- **`linkedRunners` as runner references:** Store `Set<AbstractActorRunner<?>> linkedRunners = ConcurrentHashMap.newKeySet()` in AbstractActorRunner. This avoids string ID lookups at death time and handles the case where a linked actor is deregistered before it dies.
- **Bidirectional linking:** `system.link(a, b)` adds B's runner to A's `linkedRunners` AND A's runner to B's `linkedRunners`. Symmetric.
- **Death fires LinkedActorDied to all linked runners:** In the `finally` block, after firing `Terminated` to signalListeners, iterate `linkedRunners` and call `runner.signal(new LinkedActorDied(actorId, terminalCause))` on each. Use `terminalCause` (null for graceful stop).
- **`trapExits` flag:** `volatile boolean trapExits` in AbstractActorRunner. Default `false`.
- **Non-trapped LinkedActorDied causes crash:** In `processSignalEnvelope`, if `signal instanceof LinkedActorDied && !trapExits`, throw `RuntimeException("Linked actor '" + lad.actorId() + "' died", lad.cause())`. This propagates to the loop's `catch(Exception e)`, setting `terminalCause` and triggering the actor's own death chain.
- **Trapped LinkedActorDied delivered as signal:** If `trapExits=true`, dispatch to `handleSignal(signal)` ŌåÆ actor's `onSignal(LinkedActorDied, ctx)`. Actor decides what to do.
- **`unlink`:** Remove runner references from both sides (symmetric removal). Idempotent.
- **Self-link prevention:** `link(a, a)` silently no-ops ŌĆö linking to yourself is meaningless.
- **Linked actor is already dead:** If a runner in `linkedRunners` is `!isAlive()`, skip it silently (the signal would queue into a stopped actor's mailbox and never be processed).

4. In the `finally` block, AFTER the `Terminated` notifications block (from Plan 1), add LinkedActorDied delivery:
```java
// Fire LinkedActorDied to all linked runners
if (!linkedRunners.isEmpty()) {
    LinkedActorDied exitSignal = new LinkedActorDied(actorId, terminalCause);
    for (var linked : linkedRunners) {
        try {
            if (linked.isAlive()) linked.signal(exitSignal);
        } catch (Exception ignored) {}
    }
    linkedRunners.clear();
}
```

**Modify `BayouSystem.java`:**

Add public `link` and `unlink` methods:
```java
/**
 * Bidirectionally link two actors. If either dies, the other receives a
 * {@link LinkedActorDied} signal. Unless the surviving actor is trapping exits
 * (see {@link BayouContext#trapExits}), it will also crash.
 *
 * <p>Linking to an already-stopped actor is a no-op.
 */
public void link(Ref<?> a, Ref<?> b) {
    if (a.actorId().equals(b.actorId())) return; // self-link no-op
    AbstractActorRunner<?> runnerA = runners.get(a.actorId());
    AbstractActorRunner<?> runnerB = runners.get(b.actorId());
    if (runnerA == null) throw new IllegalArgumentException("Unknown actor: " + a.actorId());
    if (runnerB == null) throw new IllegalArgumentException("Unknown actor: " + b.actorId());
    if (!runnerA.isAlive() || !runnerB.isAlive()) return; // dead actors: no-op
    runnerA.linkedRunners.add(runnerB);
    runnerB.linkedRunners.add(runnerA);
}

/** Remove a bidirectional link between two actors. Idempotent. */
public void unlink(Ref<?> a, Ref<?> b) {
    AbstractActorRunner<?> runnerA = runners.get(a.actorId());
    AbstractActorRunner<?> runnerB = runners.get(b.actorId());
    if (runnerA != null) runnerA.linkedRunners.remove(runnerB);
    if (runnerB != null) runnerB.linkedRunners.remove(runnerA);
}
```

**Add `link`/`unlink`/`trapExits` to `BayouContext<M>` interface:**
```java
/**
 * Bidirectionally link this actor to {@code other}. If either dies, the other
 * receives a {@link LinkedActorDied} signal and (unless trapping exits) also crashes.
 */
void link(Ref<?> other);

/** Remove the bidirectional link between this actor and {@code other}. Idempotent. */
void unlink(Ref<?> other);

/**
 * Set this actor's exit-trapping mode.
 * When {@code true}, {@link LinkedActorDied} signals are delivered to
 * {@link com.cajunsystems.bayou.actor.Actor#onSignal} instead of causing a crash.
 * Default is {@code false}.
 */
void trapExits(boolean flag);
```

**Implement in `BayouContextImpl<M>`:**
```java
@Override
public void link(Ref<?> other) {
    system.link(runner.toRef(), other);
}

@Override
public void unlink(Ref<?> other) {
    system.unlink(runner.toRef(), other);
}

@Override
public void trapExits(boolean flag) {
    runner.trapExits = flag;
}
```

Note: `runner.toRef()` creates a new `RefImpl` each time. Since `link()` uses runners internally (not refs for equality), this is fine. Alternatively, store a `Ref<M> selfRef` in `BayouContextImpl` ŌĆö but `toRef()` works since BayouSystem uses runner identity for link tracking.

Actually, `link(Ref<?> a, Ref<?> b)` in BayouSystem looks up by `actorId`. Since `runner.toRef()` returns a `RefImpl` with the same `actorId`, this works correctly.

### Task 2 ŌĆö Tests

Create `src/test/java/com/cajunsystems/bayou/LinkingTest.java`.

**Test cases:**

1. **`link_bothCrash_whenOneDies`** ŌĆö Link actor A and B. Crash A (throw in handler). Assert B also crashes (isAlive() becomes false, or B's postStop is called). 
   - Track: Awaitility wait for `!refB.isAlive()`.
   - The chain: A crashes ŌåÆ A's finally fires `LinkedActorDied` to B ŌåÆ B's `processSignalEnvelope` throws (no trapExits) ŌåÆ B crashes.

2. **`link_propagatesOnGracefulStop`** ŌĆö Link A and B. Call `refA.stop()`. Assert B also stops (receives `LinkedActorDied`, crashes since `terminalCause=null` causes `RuntimeException("Linked actor ... died", null)`).
   - Wait: `!refB.isAlive()`.
   - Note: LinkedActorDied with `cause=null` for graceful stop ŌĆö the thrown `RuntimeException` wraps null.

3. **`trapExits_convertsExitToSignal`** ŌĆö Actor A traps exits. Link A and B. A calls `ctx.trapExits(true)` in its `preStart`. B crashes. Assert: A is still alive (`refA.isAlive()` remains true); A's `onSignal` is called with `LinkedActorDied("B", cause)`.
   - Track: `AtomicReference<Signal>` set in A's `onSignal`.
   - Awaitility: assert signal is `LinkedActorDied`.

4. **`unlink_preventsCrashPropagation`** ŌĆö Link A and B. Then `system.unlink(refA, refB)`. Crash B. Assert A is still alive.
   - Wait 300ms after crash; assert `refA.isAlive()` is still true.

5. **`link_fromWithinActor_usingCtx`** ŌĆö Actor A links to B using `ctx.link(refB)` from within its message handler. B crashes. Assert A also crashes.

Use Awaitility for assertions. Each test uses a fresh `BayouSystem`.

**Handling the graceful-stop propagation behavior:** When a linked actor stops gracefully (`terminalCause=null`), it fires `LinkedActorDied("X", null)`. The surviving actor (no trapExits) throws `new RuntimeException("Linked actor 'X' died", null)`. This causes the surviving actor to crash. This is intentional ŌĆö matches Erlang link behavior. Document this in the test name/comment so it's explicit.

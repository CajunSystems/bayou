# Phase 10, Plan 1: PubSub / Process Groups

## Objective

Add topic-based publish/subscribe to Bayou. `system.pubsub()` returns a `BayouPubSub` registry
(one per `BayouSystem`). Actors subscribe `Ref`s to named string topics; any caller can broadcast
a message to all live subscribers on that topic. Dead actors are silently skipped and lazily
cleaned up on publish. Also adds `ctx.self()` to `BayouContext<M>` — lets actors subscribe
themselves cleanly from `preStart` or `handle` without an untyped `system.lookup()` dance.

## Context

**Current state:** `BayouSystem` has no PubSub registry. `BayouContext<M>` has no `self()` method
— actors wanting their own ref must call `ctx.system().lookup(ctx.actorId())` which returns
`Optional<Ref<?>>` (untyped, unchecked cast required). `BayouContextImpl` already holds a
`private AbstractActorRunner<M> runner` field, and `runner.toRef()` is used in the existing
`link()` implementation, so adding `self()` is a one-liner.

**Key design decisions:**

- `BayouPubSub` is a plain class (not an actor) — stateless except for the subscriptions map.
  Owned by `BayouSystem`, initialized eagerly in the field initializer.
- Subscriptions: `ConcurrentHashMap<String, CopyOnWriteArrayList<Ref<?>>>` — topic → subscriber
  list. `CopyOnWriteArrayList` makes iteration safe while subscribe/unsubscribe happen concurrently.
- `publish()` iterates, skips dead refs (`!ref.isAlive()`), swallows exceptions from `tell()` 
  (dead or overflowed actors), and does a lazy `removeIf(!isAlive())` pass after iteration to
  prevent unbounded list growth.
- Type safety: `subscribe(String topic, Ref<M>)` and `publish(String topic, M message)` are both
  generic. Internally, `((Ref<M>) ref).tell(message)` is an unchecked cast annotated with
  `@SuppressWarnings("unchecked")`. This mirrors `BayouSystem.lookup()` which already uses the
  same pattern.
- `ctx.self()` — new method on `BayouContext<M>` returning `Ref<M>`. Implemented in
  `BayouContextImpl` as `runner.toRef()`.
- No `ctx.subscribe()` shortcut — keeping PubSub access via `ctx.system().pubsub()` is
  consistent with how actors access other system facilities.

## Tasks

### Task 1 — BayouPubSub + `ctx.self()`

**Create `src/main/java/com/cajunsystems/bayou/BayouPubSub.java`:**

```java
package com.cajunsystems.bayou;

import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.CopyOnWriteArrayList;

/**
 * Topic-based publish/subscribe registry for a {@link BayouSystem}.
 *
 * <p>Obtained via {@link BayouSystem#pubsub()}. Thread-safe for concurrent
 * subscribe/publish/unsubscribe. Dead actor references are silently skipped on publish
 * and lazily cleaned up to prevent unbounded list growth.
 */
public class BayouPubSub {

    private final ConcurrentHashMap<String, CopyOnWriteArrayList<Ref<?>>> subscriptions =
            new ConcurrentHashMap<>();

    BayouPubSub() {}

    /** Subscribe {@code subscriber} to receive messages published to {@code topic}. */
    public <M> void subscribe(String topic, Ref<M> subscriber) {
        subscriptions.computeIfAbsent(topic, k -> new CopyOnWriteArrayList<>()).add(subscriber);
    }

    /** Remove {@code subscriber} from {@code topic}. No-op if not subscribed. */
    public void unsubscribe(String topic, Ref<?> subscriber) {
        var list = subscriptions.get(topic);
        if (list != null) list.remove(subscriber);
    }

    /**
     * Deliver {@code message} to all live subscribers on {@code topic}.
     *
     * <p>Dead actors are silently skipped. Bounded-mailbox overflows
     * ({@link MailboxFullException}) are silently swallowed. Dead subscribers are
     * lazily removed after each publish pass.
     */
    @SuppressWarnings("unchecked")
    public <M> void publish(String topic, M message) {
        var subscribers = subscriptions.get(topic);
        if (subscribers == null) return;
        boolean hadDead = false;
        for (Ref<?> ref : subscribers) {
            if (!ref.isAlive()) {
                hadDead = true;
                continue;
            }
            try {
                ((Ref<M>) ref).tell(message);
            } catch (Exception ignored) {}
        }
        if (hadDead) subscribers.removeIf(ref -> !ref.isAlive());
    }
}
```

**Test cases:**

**1. `multipleSubscribers_allReceivePublished`**

Three actors subscribe to topic `"news"`. Publish one message. Assert all three received it.

```java
@Test
void multipleSubscribers_allReceivePublished() {
    var received1 = new CopyOnWriteArrayList<String>();
    var received2 = new CopyOnWriteArrayList<String>();
    var received3 = new CopyOnWriteArrayList<String>();
    
    Ref<String> a = system.spawn("sub-a", (msg, ctx) -> received1.add(msg));
    Ref<String> b = system.spawn("sub-b", (msg, ctx) -> received2.add(msg));
    Ref<String> c = system.spawn("sub-c", (msg, ctx) -> received3.add(msg));
    
    var pubsub = system.pubsub();
    pubsub.subscribe("news", a);
    pubsub.subscribe("news", b);
    pubsub.subscribe("news", c);
    
    pubsub.publish("news", "hello");
    
    await().atMost(3, SECONDS).untilAsserted(() -> {
        assertThat(received1).containsExactly("hello");
        assertThat(received2).containsExactly("hello");
        assertThat(received3).containsExactly("hello");
    });
}
```

**2. `unsubscribe_stopsDelivery`**

Actor subscribes, receives first publish, unsubscribes, second publish not received.

```java
@Test
void unsubscribe_stopsDelivery() throws Exception {
    var received = new CopyOnWriteArrayList<String>();
    Ref<String> actor = system.spawn("unsub", (msg, ctx) -> received.add(msg));
    
    var pubsub = system.pubsub();
    pubsub.subscribe("updates", actor);
    pubsub.publish("updates", "first");
    await().atMost(3, SECONDS).untilAsserted(() -> assertThat(received).hasSize(1));
    
    pubsub.unsubscribe("updates", actor);
    pubsub.publish("updates", "second");
    
    // Give time for any spurious delivery — count must stay at 1
    Thread.sleep(200);
    assertThat(received).hasSize(1).containsExactly("first");
}
```

**3. `deadActor_silentlySkippedOnPublish`**

Two actors subscribe. Stop one. Publish. Assert no exception; the live actor still receives.

```java
@Test
void deadActor_silentlySkippedOnPublish() throws Exception {
    var received = new CopyOnWriteArrayList<String>();
    Ref<String> alive = system.spawn("alive", (msg, ctx) -> received.add(msg));
    Ref<String> dead  = system.spawn("dead",  (msg, ctx) -> {});
    
    var pubsub = system.pubsub();
    pubsub.subscribe("events", alive);
    pubsub.subscribe("events", dead);
    
    dead.stop().get(2, SECONDS);
    
    // Must not throw even though "dead" is stopped
    assertThatCode(() -> pubsub.publish("events", "hi")).doesNotThrowAnyException();
    
    await().atMost(3, SECONDS).untilAsserted(() -> assertThat(received).containsExactly("hi"));
}
```

**4. `topicIsolation_onlyCorrectTopicReceives`**

**5. `ctxSelf_allowsInActorSubscription`**

Actor subscribes itself via `ctx.self()` in `preStart`. Verify it receives published messages.

```java
@Test
void ctxSelf_allowsInActorSubscription() {
    var received = new CopyOnWriteArrayList<String>();
    
    system.spawn("self-sub", new Actor<String>() {
        @Override
        public void handle(String msg, BayouContext<String> ctx) {
            received.add(msg);
        }
        @Override
        public void preStart(BayouContext<String> ctx) {
            ctx.system().pubsub().subscribe("greet", ctx.self());
        }
    });
    
    await().atMost(1, SECONDS).until(() -> system.lookup("self-sub").isPresent());
    system.pubsub().publish("greet", "hello-self");
    
    await().atMost(3, SECONDS).untilAsserted(() -> assertThat(received).containsExactly("hello-self"));
}
```

Note: The `await().until(() -> system.lookup("self-sub").isPresent())` is NOT sufficient — the actor might be registered in `actors` before `preStart` finishes. Use a small `await` on `received` being non-empty only after publish, OR add a brief sleep after spawn. Actually, looking at how Bayou works: `runner.start()` is called in `spawn()`, which starts the virtual thread. `preStart()` is called from within `initialize()` which is the first thing the virtual thread does. The actor is registered in `actors` before `start()` returns. However, `preStart()` runs asynchronously on the virtual thread. So there's a race: publish could happen before `preStart` subscribes.

Fix: Use an `ask` to ensure the actor has processed at least one message (which means `preStart` has already run since it runs before the message loop). Or add a `CompletableFuture<Void>` latch that `preStart` completes.

Simpler fix: use a `CountDownLatch` that `preStart` counts down:

```java
@Test
void ctxSelf_allowsInActorSubscription() throws Exception {
    var received = new CopyOnWriteArrayList<String>();
    var preStartDone = new CountDownLatch(1);
    
    system.spawn("self-sub", new Actor<String>() {
        @Override
        public void handle(String msg, BayouContext<String> ctx) {
            received.add(msg);
        }
        @Override
        public void preStart(BayouContext<String> ctx) {
            ctx.system().pubsub().subscribe("greet", ctx.self());
            preStartDone.countDown();
        }
    });
    
    preStartDone.await(2, TimeUnit.SECONDS);
    system.pubsub().publish("greet", "hello-self");
    
    await().atMost(3, SECONDS).untilAsserted(() -> assertThat(received).containsExactly("hello-self"));
}
```

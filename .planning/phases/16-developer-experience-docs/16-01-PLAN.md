### Task 2 — CHANGELOG.md + docs/getting-started.md

**Create `CHANGELOG.md`** at the project root:

```markdown
# Changelog

All notable changes to this project will be documented in this file.

## [0.1.0] — 2026-04-17

### Added

#### Milestone 1: Supervision
- `SupervisorActor` — declare child actors and a supervision strategy
- `OneForOneStrategy` / `AllForOneStrategy` — restart policies with `RestartWindow`
- `RestartDecision` — `RESTART`, `RESTART_ALL`, `STOP`, `ESCALATE`
- Death spiral guard — supervisor escalates after exceeding restart window
- Nested supervisors — multi-level supervision trees
- Dynamic children — `SupervisorRef.spawnChild(ChildSpec)` at runtime

#### Milestone 2: Erlang/Elixir Feature Parity
- Timer messages — `ctx.scheduleOnce(Duration, M)` / `ctx.schedulePeriodic(Duration, M)` / `TimerRef.cancel()`
- Death watch — `system.watch(target, watcher)` / `ctx.watch(target)` → `Terminated` signal
- Actor linking — `system.link(a, b)` / `ctx.link(ref)` → `LinkedActorDied` exit signal
- Exit trapping — `ctx.trapExits(true)` converts exit signals to deliverable signals
- Back-pressure — `MailboxConfig.bounded(int)` with `DROP_NEWEST`, `DROP_OLDEST`, `REJECT` strategies
- `OverflowListener` — observability hook for mailbox overflow events
- PubSub — `system.pubsub()` returns `BayouPubSub`; `ctx.self()` for in-actor subscription
- `StateMachineActor<S, M>` — FSM behavior with `onEnter`/`onExit` callbacks

#### Milestone 3: Persistent PubSub & Developer Experience
- `BayouTopic<M>` — durable log-backed topic via `system.topic(name, serializer)`
- Durable subscriptions — `topic.subscribe(subscriptionId, ref)` tracks position; `topic.unsubscribeDurable(id)` forgets it
- Message replay — `topic.subscribeFrom(offset, ref)` / `topic.subscribeFromBeginning(ref)` / `topic.latestOffset()`
- `TestKit` / `TestProbe<M>` — synchronous actor testing; `expectMessage`, `expectNoMessage`, `expectTerminated`
- gumbo 0.2.0 — `readAfter`, `getLatestSeqnum`, `setValue/getValue`, push subscriptions

### Changed
- `BayouContext<M>` is now generic — type-safe timer and self-reference APIs
- `Ref<M>` replaces `ActorRef<M>` in all public APIs

[0.1.0]: https://github.com/CajunSystems/bayou/releases/tag/0.1.0
```

8. **Test with TestKit** — `TestProbe<String> probe = TestKit.probe(system, "probe")`, `probe.ref().tell(...)`, `probe.expectMessage(...)`, `probe.expectTerminated(...)`
9. **What's next** — links to `actor-flavours.md`, `testing.md`, `persistent-pubsub.md`, `supervision.md`, `patterns.md`

### Task 3 — docs/actor-flavours.md + docs/testing.md

**Create `docs/actor-flavours.md`:**

Guide for choosing the right actor flavor. Structure:
1. **Quick decision table** — 4-column table: Actor, StatefulActor, EventSourcedActor, StateMachineActor — rows: gumbo usage, restart behavior, memory footprint, use case
2. **Actor (stateless)** — description, when to use (pure computation, I/O, message routing, most actors), code example
3. **StatefulActor (snapshot-based)** — description, when to use (counters, caches, aggregates where recent state matters more than full history), snapshot mechanics, code example
4. **EventSourcedActor (log-sourced)** — description, when to use (financial ledgers, audit logs, anything that needs full event history or time travel), replay mechanics, code example
5. **StateMachineActor (FSM)** — description, when to use (connection lifecycle, order workflow, multi-step processes with defined transitions), callback semantics, code example
6. **Mixed architectures** — supervisor that coordinates different flavors; link patterns
7. **In doubt?** — default recommendation: start with `Actor`, upgrade to `EventSourcedActor` if you need durability

Target length: ~200 lines.

**Create `docs/testing.md`:**

Guide for testing actors. Structure:
1. **TestKit setup** — `TestKit.probe(system, "probe")` in a `@BeforeEach`; the three key methods
2. **Basic message assertion** — `probe.ref().tell(msg)` + `probe.expectMessage(msg, Duration.ofSeconds(1))` — done-when example
3. **Type-based assertion** — `probe.expectMessage(String.class, ...)` for sealed interface messages
4. **Asserting silence** — `probe.expectNoMessage(Duration.ofMillis(200))` — use for negative tests
5. **Watching for actor death** — `probe.expectTerminated(ref, Duration.ofSeconds(2))` — after stopping an actor
6. **PubSub with TestKit** — subscribe probe to a topic, publish, assert
7. **Testing with Awaitility** — when TestKit isn't enough (timing-sensitive, distributed assertions); `Awaitility.await().atMost(...)` pattern still in the codebase
8. **Tips** — use unique actor IDs per test; always shut down the system in `@AfterEach`; use `InMemoryPersistenceAdapter` for tests

Target length: ~150 lines.
